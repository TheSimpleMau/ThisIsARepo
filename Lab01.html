Preguntas de tarea

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es un sistema de redes conectadas a escala mundial que permite la comunicación y el acceso a los datos. Se rige por organismos como la IANA (Intenet Assinged Numbers Authority), que establecen protocolos universales.

Por otro lado, la World Wide Web (por sus siglas www), es el principal servicio de recuperación de información en la red mundial. Permite a los usarios acceder a una gran colección de documentos dentro de otras computadoras/servidores.

¿Cuáles son las partes de un URL?

URL es el acrónimo de Uniform Resource Locator y se compone de cuatro componentes principales:

  1. Protocolo: La mayoría es HTTP/HTTPS, pero también existen los protocolos FTP y SMTP.
  2. Dominio: El nombre con el que se identifíca una o más direcciones IP dónde el recurso está localizado.
  3. Ruta: Especifíca la localización del recurso dentro del servidor.
  4. Parámetros: Datos adicionales para identificar o flitrar el recurso del servidor.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Sirve para pedirle algo a un servidor. No tiene cuerpo. HEAD: Pide una respuesta idéntica a la de GET, pero sin cuerpo. Sirve más para realizar diagnosticos. POST: Sirve para enviarle algo al serivdor. Dentro de este método el cuerpo va cifrado. PUT: Sirve para actualizar o reemplazar completamente un recurso en el servidor. PATCH: Sirve para actualizar solamente una parte de un recurso. DELETE: Se utiliza para eliminar un recurso del servidor.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El método HTTP que se utiliza cuando se accede a una página web a través de una URL es GET. Esto pues GET se utiliza para solicitar recursos como, mismamente, una página HTML.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Cuándo un servidor web devuelve una respuesta HTTP de código 200 signifíca que todo ha salido bien, es decir, que existe el recurso.

No, pues podría ser que el usuario esté intentando de acceder a un recurso del cuál no debería de existir. Aunque, en ocasiones, si el recurso sí que existía pero se cambió de lugar, entonces podría reportarse.

Sí, ya que un error 500 significa que el problema es estrictamente con nuestro servidor.

¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla se estructura con una serie de elementos que definenen su composición y permiten aplicar estilos de manera efectiva. Los principales componentes de estructura y estilo para una tabla se presentan a continuación.

De estructura:
  • <table>: Es el elemento contenedor que define la tabla en su totalidad.
  • <caption>: Proporciona un título o descripción para la tabla.
  • <thead>: Agrupa las filas que conforman el encabezado de la tabla.
  • <tbody>: Contiene las filas que representan el cuerpo principal de la tabla.
  • <tfoot>: Agrupa las filas que constituyen el pie de la tabla, útil para notas finales.
  • <tr>: Define una fila dentro de la tabla.
  • <th>: Especifica una celda de encabezado dentro de una fila, generalmente utilizada para títulos de columnas o filas.
  • <td>: Define una celda de datos estándar dentro de una fila.
  • <colgroup>: Permite agrupar una o más columnas para aplicar estilos o atributos compartidos.
  • <col>: Especifica propiedades para una o varias columnas dentro de un <colgroup>.

Mencionar que, para las opciones de estilo, estas principalmente se manejan dentro de CSS. Así pues, estas son los componentes de estilo:

  • border: Define el estilo, grosor y color de los bordes de la tabla y sus celdas.
  • border-collapse: Determina si los bordes de la tabla y las celdas se muestran separados o colapsados en un solo borde.
  • padding: Establece el espacio interior entre el contenido de una celda y su borde.
  • text-align: Alinea horizontalmente el contenido dentro de las celdas.
  • vertical-align: Alinea verticalmente el contenido dentro de las celdas.
  • background-color: Aplica un color de fondo a la tabla, filas o celdas específicas.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

Este ciclo abarca todas las fases que experimenta un sistema de información a lo largo de su existencia. Las etapas principales son:

  • La planificación: Donde se realiza una serie de tareas previas que influirán en la finalización del proyecto.
  • Análisis: Es la parte en donde se avergiua qué es exactamente lo que tiene que hacer el sistema.
  • Diseño: Se estudian posibles alternativas de implementación para el sistema de información para poder construirlo.
  • implementación: Se selecionan las herramientas adecuadas para que sea un entorno de desarrollo que facilíte el trabajo.
  • Pruebas: Aquí se detectan los posibles errores que se hayan podido comenter en los pasos anteriores.
  • Instalación o despliegue: Se planifica el entorno en el que el sistema debe de funcionar, tanto en hardware cómo en software.
  • Uso y mantenimieno: Aquí el principal objetivo es:
    • Eliminar los defectos.
    • Adaptarlo a nuevas necesidades.
    • Añadirle nuevas funcionalidades.

¿Cuál es el ciclo de desarrollo de sistemas de información?

Este ciclo se enfoca específicamente en el proceso de creación y evolución del sistema. Las etapas clave incluyen:

  • Planificación: Definición de objetivos, alcance y recursos necesarios para el desarrollo del sistema.
  • Análisis: Determinación de los requisitos funcionales y no funcionales del sistema.
  • Diseño: Elaboración de la estructura y componentes del sistema, incluyendo interfaces y bases de datos.
  • Desarrollo: Codificación y construcción de los componentes del sistema.
  • Pruebas: Evaluación del sistema para asegurar su correcto funcionamiento y cumplimiento de requisitos.
  • Implementación: Integración del sistema en el entorno operativo y formación de los usuarios.
  • Mantenimiento: Actualización y mejora continua del sistema para adaptarse a nuevas necesidades o corregir errores.

Si bien ambos son muy similares, el cilco de vida de los sistemas de información tiene un enfoque más amplio, abarcando desde la concepción hasta el retiro del sistema. Mientras que el cliclo de desarrollo de sistemas de información se centra más en etapas específicas de creación y evolución de un sistema.
