¿Qué diferencias y semejanzas hay entre Java y JavaScript

En realidad, Java y JavaScript actualmente son muy diferntes entre sí. En los inicios de JavaScript pudiese ser que existien cosas similares entre sí pero, actualmente, fuera de que en su sintaxis ambos usan llaves y puntos y comas y que los dos son lenguajes orientado a objetos, no existe mayor semejanza.

Principalmente, podemos ver sus diferencias en sus propositos, pues Java esta 100% enfocado en la programación orientada a objetos, mientras que, si bien JavaScript también lo es, en realidad es multiparadigma. Además, en su ejecución de Java se utiliza la Java Virtual Machine para ejecutrase, mientras que JavaScript se puede ejecutra dentro de cualquier navegador. El tipado de Java es más fuerte que el de JavaScript. En resumen existen muchisimas diferencias entre Java y JavaScript.

¿Qué métodos tiene el objeto Date? (Menciona al menos 5*)

Principalmente podríamos divirlos entre dos grandes componentes, get y set, además de que existe cada uno para Local y para UTC. Como ejemplos, tenemos getFullYear() y getUTCFullYear().

getFullYear() regresa el año de una fecha de acuerdo al tiempo local, mientras que getUTCFullYear() lo regresa de acuerdo al tiempo universal.

Así pues, como ejemplo podríamos tener los siguientes métodos:

  • getMonth(): regresa el mes de una fecha, como un valor.
  • getDate(): regresa el día del mes (número).
  • getDay(): regresa el día de la semana como un valor, el 0 es el domingo.
  • getHours(): regresa la hora de una fecha.
  • getMinutes(): regresa el minuto de una fecha.
En general, existe un método para cada momento de una fecha (hasta milisegundos) y esta se puede ajustar al tiempo local o al tiempo UTC. Además de que se puede modificar con set.

¿Qué métodos tienen los arreglos? (Menciona al menos 5*)

Los arreglos tiene principalmente los siguientes métodos:

  • .push(elemento): nos sirve para añadir un elemento al final del arreglo.
  • .pop(): elimina el último elemento del arreglo y lo regresa.
  • .shift(): elimina el primer elemento del arreglo y lo regresa.
  • .unshift(elemento): agrega un elemento al inicio del arreglo.
  • .map(funcion): crea un nuevo arreglo aplicandole una función a cada elemento.

¿Cómo se declara una variable con alcance local dentro de una función?

Se declara dentro de la función con let o con const, por ejemplo:

            
                function miFuncion(){
                    let var1 = 1; //Esto es una variable local
                    const var2 = 2; //Esto es una constante local
                    return 0;
                }

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Como ventaja, podemos utilizar esta variable dentro de cualquier función. Por ejemplo, si quicieramos definir por nuestra cuenta el valor de π. Sin embargo si en algún punto de nuestra función cambiamos el valor de la variable, esta se verá afectada por lo que nos podría costar trabajo saber cual es el valor real de esta variable durante todo el código.
